# 部署教程

将 AI 叙事跑团游戏部署到云服务器的完整指南。

---

## 服务器要求

| 项目 | 最低 | 推荐 |
|------|------|------|
| CPU | 2 核 | 4 核 |
| 内存 | 4 GB | 8 GB |
| 硬盘 | 40 GB SSD | 80 GB SSD |
| 系统 | Ubuntu 22.04 | Ubuntu 22.04 |

推荐：阿里云 ECS、腾讯云 CVM、AWS EC2

---

## 一键部署

SSH 登录服务器后执行：

```bash
# 上传代码到服务器（二选一）
git clone https://github.com/your-repo/ai-rpg.git /home/deploy/ai-rpg
# 或 scp -r ./project/* user@server:/home/deploy/ai-rpg/

cd /home/deploy/ai-rpg

# 配置环境变量
cp .env.example .env
vim .env   # 填入 AI API Key、数据库密码等

# 运行部署脚本
chmod +x deploy.sh
./deploy.sh
```

## 手动部署（分步）

### 第 1 步：安装 Docker

```bash
curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker $USER
# 重新登录使权限生效
```

### 第 2 步：配置环境变量

```bash
cp .env.example .env
vim .env
```

必填项：

```bash
# 数据库密码（生成一个强密码）
POSTGRES_PASSWORD=your-strong-password

# AI 配置
AI_API_KEY=sk-your-api-key
AI_BASE_URL=https://api.openai.com/v1
AI_MODEL_DEFAULT=gpt-4o-mini
AI_MODEL_PREMIUM=gpt-4o

# JWT 密钥（生成：openssl rand -hex 32）
JWT_SECRET_KEY=your-jwt-secret

# 前端访问地址
NEXT_PUBLIC_API_URL=http://your-server-ip
```

### 第 3 步：构建启动

```bash
docker compose -f docker-compose.prod.yml up -d --build
```

### 第 4 步：初始化数据库

```bash
docker compose -f docker-compose.prod.yml exec backend alembic upgrade head
```

### 第 5 步：验证

```bash
# 检查服务状态
docker compose -f docker-compose.prod.yml ps

# 测试 API
curl http://localhost:8000/health
```

浏览器访问 `http://your-server-ip` 即可。

## 域名 + HTTPS

### 1. DNS 解析

添加 A 记录，指向服务器 IP。

### 2. 申请 SSL 证书

```bash
sudo apt install certbot
sudo certbot certonly --standalone -d your-domain.com

# 复制证书
mkdir -p nginx/ssl
sudo cp /etc/letsencrypt/live/your-domain.com/fullchain.pem nginx/ssl/
sudo cp /etc/letsencrypt/live/your-domain.com/privkey.pem nginx/ssl/
```

### 3. 更新配置

编辑 `.env`：
```bash
NEXT_PUBLIC_API_URL=https://your-domain.com
```

编辑 `nginx/nginx.conf`，启用 HTTPS server block，然后重启：

```bash
docker compose -f docker-compose.prod.yml restart nginx
```

## 日常运维

```bash
# 查看日志
docker compose -f docker-compose.prod.yml logs -f

# 重启服务
docker compose -f docker-compose.prod.yml restart

# 更新部署
git pull
docker compose -f docker-compose.prod.yml up -d --build
docker compose -f docker-compose.prod.yml exec backend alembic upgrade head

# 停止服务
docker compose -f docker-compose.prod.yml down
```

### 数据库备份

```bash
# 手动备份
docker compose -f docker-compose.prod.yml exec postgres \
  pg_dump -U gameuser gamedb > backup_$(date +%Y%m%d).sql

# 恢复
docker compose -f docker-compose.prod.yml exec -T postgres \
  psql -U gameuser gamedb < backup_20260429.sql
```

自动备份已配置在 `deploy.sh` 中（每天凌晨 3 点）。

## 常见问题

**服务启动失败**
```bash
docker compose -f docker-compose.prod.yml logs backend
# 通常是 .env 配置错误或端口占用
```

**AI 调用失败**
```bash
# 检查 API 配置
docker compose -f docker-compose.prod.yml exec backend env | grep AI_
# 检查网络连通性
docker compose -f docker-compose.prod.yml exec backend curl -I https://api.openai.com
```

**数据库连接失败**
```bash
docker compose -f docker-compose.prod.yml ps postgres
docker compose -f docker-compose.prod.yml logs postgres
```

**磁盘空间不足**
```bash
docker system prune -a   # 清理 Docker 缓存
```
