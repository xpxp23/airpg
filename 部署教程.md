# 部署教程

将 AI 叙事跑团游戏部署到云服务器的完整指南。

仓库地址：https://github.com/xpxp23/airpg

---

## 服务器要求

| 项目 | 最低 | 推荐 |
|------|------|------|
| CPU | 2 核 | 4 核 |
| 内存 | 4 GB | 8 GB |
| 硬盘 | 40 GB SSD | 80 GB SSD |
| 系统 | Ubuntu 22.04 / Debian 12 | Ubuntu 22.04 |

推荐云服务商：阿里云 ECS、腾讯云 CVM、AWS EC2、Vultr

> 需要开放端口：80（HTTP）、443（HTTPS，可选）、22（SSH）

---

## 方式一：一键部署（推荐）

SSH 登录服务器后，执行以下命令即可完成全部部署：

```bash
# 1. 克隆项目
git clone https://github.com/xpxp23/airpg.git /opt/airpg
cd /opt/airpg

# 2. 配置环境变量（必填）
cp .env.example .env
vim .env   # 详见下方"环境变量说明"

# 3. 运行部署脚本
chmod +x deploy.sh
./deploy.sh              # 海外服务器
./deploy.sh --mirror     # 国内服务器（自动配置阿里云镜像加速）
```

> 国内服务器建议加 `--mirror` 参数，会自动配置 Docker/npm/pip 国内镜像源，大幅加速构建。

脚本会自动完成：
- Docker 安装检测
- 服务构建与启动
- 数据库初始化
- 每日自动备份配置

部署完成后访问 `http://你的服务器IP` 即可。

## 方式二：手动部署

适合需要自定义配置或排查问题的场景。

### 第 1 步：安装 Docker

```bash
# 安装 Docker
curl -fsSL https://get.docker.com | sudo sh

# 将当前用户加入 docker 组（免 sudo）
sudo usermod -aG docker $USER

# 重新登录使权限生效，或执行：
newgrp docker

# 验证安装
docker --version
docker compose version
```

### 第 2 步：获取代码

```bash
git clone https://github.com/xpxp23/airpg.git /opt/airpg
cd /opt/airpg
```

### 第 3 步：配置环境变量

```bash
cp .env.example .env
vim .env
```

**必填项说明：**

```bash
# ---------- 数据库 ----------
POSTGRES_DB=gamedb
POSTGRES_USER=gameuser
POSTGRES_PASSWORD=替换为强密码          # 必须修改

# ---------- AI 配置 ----------
AI_PROVIDER=openai                     # openai / anthropic / local / custom
AI_API_KEY=sk-你的API密钥              # 必须修改
AI_BASE_URL=https://api.openai.com/v1  # 替换为你的API地址
AI_MODEL_DEFAULT=gpt-4o-mini           # 低成本任务模型
AI_MODEL_PREMIUM=gpt-4o                # 叙事生成模型

# ---------- JWT ----------
JWT_SECRET_KEY=替换为随机密钥           # 生成命令: openssl rand -hex 32

# ---------- 前端 ----------
NEXT_PUBLIC_API_URL=http://你的服务器IP  # 生产环境填域名(https://your.com)
```

**AI 服务配置示例：**

| 服务 | AI_BASE_URL | AI_MODEL_DEFAULT | AI_MODEL_PREMIUM |
|------|-------------|------------------|------------------|
| OpenAI 官方 | `https://api.openai.com/v1` | `gpt-4o-mini` | `gpt-4o` |
| DeepSeek | `https://api.deepseek.com/v1` | `deepseek-chat` | `deepseek-chat` |
| 通义千问 | `https://dashscope.aliyuncs.com/compatible-mode/v1` | `qwen-turbo` | `qwen-plus` |
| Moonshot | `https://api.moonshot.cn/v1` | `moonshot-v1-8k` | `moonshot-v1-32k` |
| 智谱 GLM | `https://open.bigmodel.cn/api/paas/v4` | `glm-4-flash` | `glm-4` |
| 零一万物 | `https://api.lingyiwanwu.com/v1` | `yi-lightning` | `yi-large` |
| 本地 Ollama | `http://localhost:11434/v1` | `qwen2.5:7b` | `qwen2.5:14b` |
| 本地 vLLM | `http://localhost:8000/v1` | 自选模型 | 自选模型 |

### 第 4 步：构建启动

```bash
# 构建镜像并启动所有服务
docker compose -f docker-compose.prod.yml up -d --build

# 等待服务就绪（约 30 秒）
sleep 30

# 检查服务状态（全部应为 running）
docker compose -f docker-compose.prod.yml ps
```

### 第 5 步：初始化数据库

```bash
docker compose -f docker-compose.prod.yml exec backend alembic upgrade head
```

### 第 6 步：验证部署

```bash
# 测试后端 API
curl http://localhost:8000/health

# 测试前端
curl -I http://localhost

# 测试注册接口
curl -X POST http://localhost:8000/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"test","email":"test@test.com","password":"123456"}'
```

浏览器访问 `http://你的服务器IP` 即可看到登录页面。

## 域名 + HTTPS

### 1. DNS 解析

在域名服务商处添加 A 记录：

| 主机记录 | 类型 | 记录值 |
|---------|------|--------|
| @ | A | 服务器 IP |
| www | A | 服务器 IP |

### 2. 申请 SSL 证书

```bash
# 安装 certbot
sudo apt update && sudo apt install -y certbot

# 申请证书（需先停止 nginx 占用 80 端口）
docker compose -f docker-compose.prod.yml stop nginx
sudo certbot certonly --standalone -d your-domain.com -d www.your-domain.com

# 复制证书到项目目录
mkdir -p nginx/ssl
sudo cp /etc/letsencrypt/live/your-domain.com/fullchain.pem nginx/ssl/
sudo cp /etc/letsencrypt/live/your-domain.com/privkey.pem nginx/ssl/
sudo chown -R $USER:$USER nginx/ssl/
```

### 3. 更新 Nginx 配置

编辑 `nginx/nginx.conf`，将 HTTP server block 替换为 HTTPS：

```nginx
server {
    listen 80;
    server_name your-domain.com www.your-domain.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name your-domain.com www.your-domain.com;

    ssl_certificate     /etc/nginx/ssl/fullchain.pem;
    ssl_certificate_key /etc/nginx/ssl/privkey.pem;
    ssl_protocols       TLSv1.2 TLSv1.3;

    # ... 其余 location 配置保持不变 ...
}
```

### 4. 更新环境变量并重启

```bash
# 更新前端 API 地址
sed -i 's|NEXT_PUBLIC_API_URL=.*|NEXT_PUBLIC_API_URL=https://your-domain.com|' .env

# 重新构建前端（环境变量需在构建时注入）
docker compose -f docker-compose.prod.yml up -d --build frontend nginx
```

浏览器访问 `https://your-domain.com` 验证。

## 日常运维

### 常用命令

```bash
# 查看所有服务状态
docker compose -f docker-compose.prod.yml ps

# 查看实时日志
docker compose -f docker-compose.prod.yml logs -f

# 查看某个服务日志
docker compose -f docker-compose.prod.yml logs -f backend

# 重启所有服务
docker compose -f docker-compose.prod.yml restart

# 重启单个服务
docker compose -f docker-compose.prod.yml restart backend

# 停止所有服务
docker compose -f docker-compose.prod.yml down

# 启动所有服务
docker compose -f docker-compose.prod.yml up -d
```

### 更新部署

```bash
cd /opt/airpg

# 拉取最新代码
git pull

# 重新构建并启动
docker compose -f docker-compose.prod.yml up -d --build

# 执行数据库迁移（如有新迁移）
docker compose -f docker-compose.prod.yml exec backend alembic upgrade head
```

### 数据库备份

```bash
# 手动备份
docker compose -f docker-compose.prod.yml exec postgres \
  pg_dump -U gameuser gamedb | gzip > backup_$(date +%Y%m%d_%H%M%S).sql.gz

# 恢复备份
gunzip < backup_20260429_120000.sql.gz | \
  docker compose -f docker-compose.prod.yml exec -T postgres \
  psql -U gameuser gamedb
```

自动备份已在 `deploy.sh` 中配置（每天凌晨 3 点），备份文件在 `/home/用户名/backups/` 目录。

### 资源监控

```bash
# 查看容器资源占用
docker stats --no-stream

# 查看磁盘使用
df -h
docker system df

# 清理无用镜像和缓存
docker system prune -a
```

## 常见问题

### 服务启动失败

```bash
# 查看具体错误
docker compose -f docker-compose.prod.yml logs backend

# 常见原因：
# 1. .env 配置错误（缺少必填项、格式错误）
# 2. 端口被占用（80/8000/5432/6379）
# 3. 内存不足
```

### AI 调用失败

```bash
# 检查环境变量是否正确注入
docker compose -f docker-compose.prod.yml exec backend env | grep AI_

# 测试 API 连通性
docker compose -f docker-compose.prod.yml exec backend \
  python -c "import httpx; r=httpx.get('https://api.openai.com/v1/models', headers={'Authorization':'Bearer YOUR_KEY'}); print(r.status_code)"

# 常见原因：
# 1. API Key 无效或过期
# 2. BASE_URL 地址错误
# 3. 服务器网络无法访问 API（需要代理）
```

### 数据库连接失败

```bash
# 检查 postgres 状态
docker compose -f docker-compose.prod.yml ps postgres
docker compose -f docker-compose.prod.yml logs postgres

# 常见原因：
# 1. POSTGRES_PASSWORD 与 DATABASE_URL 中的密码不一致
# 2. 数据卷损坏（需删除 pgdata 卷重建）
```

### 前端白屏

```bash
# 检查前端容器日志
docker compose -f docker-compose.prod.yml logs frontend

# 常见原因：
# 1. NEXT_PUBLIC_API_URL 配置错误
# 2. 后端服务未启动
# 3. 需要重新构建前端（环境变量在构建时注入）
docker compose -f docker-compose.prod.yml up -d --build frontend
```

### 端口被占用

```bash
# 查看占用端口的进程
sudo lsof -i :80
sudo lsof -i :5432

# 停止占用进程或修改 docker-compose.prod.yml 中的端口映射
```

### 内存不足

```bash
# 查看内存使用
free -h

# 创建 swap（适用于 2G/4G 小内存机器）
sudo fallocate -l 4G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

## 生产环境建议

1. **定期更新密码**：数据库密码、JWT 密钥、API Key 定期轮换
2. **限制访问**：使用防火墙只开放 22/80/443 端口（`ufw`）
3. **日志收集**：配置日志外发到 ELK/Loki 等平台
4. **监控告警**：配置服务器监控（Prometheus/Grafana 或云厂商监控）
5. **数据库备份**：除自动备份外，定期将备份文件同步到对象存储
